Versión Web de QRGB

Información importante

Esta versión de QRGB está optimizada para ejecutarse directamente en un navegador web, lo que la hace perfecta para alojarla en GitHub Pages u otros servicios de hosting estático.

Implementación en GitHub Pages

Pasos para publicar la aplicación:

  1. Crea un nuevo repositorio en GitHub
  2. Sube todos los archivos incluidos en el paquete al repositorio
  3. Ve a la sección Settings del repositorio
  4. Navega a Pages en el menú lateral
  5. En "Source", selecciona la rama main (o master)
  6. Haz clic en Save para iniciar el despliegue
  7. Tu aplicación estará disponible en https://[tu-usuario].github.io/[nombre-repo]

Personalización

Puedes personalizar la apariencia y comportamiento de la aplicación web:

Modifica el archivo styles.css para cambiar colores, fuentes y disposición:

/* Ejemplo: Cambiar colores principales */
:root {
  --primary-color: #3498db;  /* Azul */
  --secondary-color: #2ecc71;  /* Verde */
  --accent-color: #e74c3c;  /* Rojo */
}

/* Personalizar encabezado */
header {
  background-color: var(--primary-color);
  color: white;
}

/* Personalizar botones */
.btn-custom {
  background-color: var(--secondary-color);
  border: none;
}

El archivo qrgb.js contiene la implementación principal del generador:

// Ejemplo: Modificar tamaño predeterminado del QR
const DEFAULT_QR_SIZE = 300; // Cambiar según necesidad

// Ejemplo: Añadir nuevo nivel de corrección de errores
function setErrorCorrectionLevel(level) {
  switch(level) {
    case 'L': return { errorCorrectionLevel: 'L' }; // 7%
    case 'M': return { errorCorrectionLevel: 'M' }; // 15%
    case 'Q': return { errorCorrectionLevel: 'Q' }; // 25%
    case 'H': return { errorCorrectionLevel: 'H' }; // 30%
    case 'MAX': // Nivel personalizado (ejemplo)
      return { 
        errorCorrectionLevel: 'H',
        maskPattern: 4
      };
    default:
      return { errorCorrectionLevel: 'M' };
  }
}

Preguntas frecuentes

Sí, QRGB está disponible bajo una licencia libre que permite su uso comercial. Solo debes mantener la atribución al proyecto original en los comentarios del código.

Una vez cargada la página, la aplicación funciona completamente en el navegador. Sin embargo, se requiere conexión para las dependencias externas como Bootstrap y las librerías de códigos QR la primera vez que se accede.

Los códigos individuales de cada capa (rojo, verde y azul) son compatibles con cualquier aplicación de escaneo de QR. El código combinado RGB puede ser más difícil de escanear y generalmente requiere filtros de color para acceder a cada capa individualmente.

Puedes reportar problemas y sugerir mejoras en la sección de Issues del repositorio de GitHub. También puedes contactar directamente al desarrollador a través del correo electrónico proporcionado en el archivo README.
